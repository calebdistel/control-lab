## Hook

Picture this: you tell your friend, "make me a sandwich." They walk into the kitchen and come back two minutes later with a sandwich. Easy.

Now imagine your friend is a robot. You say "make me a sandwich." The robot stands there. You sigh, and try again: "Pick up the bread." The robot picks up the entire loaf — wrapper and all — and hands it to you.

Programming a robot is exactly this conversation. Every instruction you'd skip when talking to a human, you have to say out loud to the machine.

## Core concept

<Key>
A program is a sequence of instructions, executed in order, by something that does exactly what you say — no more, no less.
</Key>

That last part is the trap. A computer will do precisely what your program tells it to. If your program is wrong in a subtle way, the robot will be subtly wrong, confidently, fifty times a second.

## Walk-through

The task: get a robot arm from resting at 0° to holding a game piece at 90°.

A first attempt:

> Move the arm to 90°.

That's not a program — that's a goal. A program would look more like:

1. Read the current arm angle from the sensor.
2. If the angle is less than 90°, run the motor up.
3. If the angle is more than 90°, run the motor down.
4. If the angle is close enough to 90°, stop the motor.
5. Go back to step 1.

Notice three things. Every step is something the robot can **actually do** — "move to 90°" isn't one of those things, but "run the motor" is. There's a **loop** — the robot does this ~50 times per second, forever. And there's a **decision** — "if… then…" is how a program responds to the world.

This five-line program is, almost literally, your first controller. Step through it once here to see exactly what happens on one iteration when `angle` starts at 45°:

<CodeTrace
  client:load
  variables={{ angle: 45, motor: "off" }}
  steps={[
    { code: "angle = sensor.read()", state: { angle: 45, motor: "off" }, comment: "read hardware" },
    { code: "if angle < 90 → motor UP", state: { angle: 45, motor: "up" }, comment: "condition true" },
    { code: "if angle > 90 → motor DOWN", state: { angle: 45, motor: "up" }, comment: "condition false" },
    { code: "if |angle - 90| < 2 → STOP", state: { angle: 45, motor: "up" }, comment: "condition false" },
    { code: "→ go back to step 1", state: { angle: 47, motor: "up" }, comment: "loop!" },
  ]}
/>

The motor ran for one cycle and `angle` ticked up by 2°. On the next iteration it checks again. This is how all robot control works.

## Key takeaways

- A program is a sequence of instructions executed in order.
- Computers do exactly what you say — including your mistakes.
- Robot programs loop continuously, reading sensors and deciding what to do.
- "Move the arm to 90°" is a goal. "Run the motor while angle < 90°" is a program.

## Common confusions

**"This seems too simple."** It is, on purpose. The reason students struggle later isn't that programs get conceptually complicated — it's that they get *long*. The discipline of being explicit is what we're building here.

**"Won't the robot figure out what I mean?"** No. Every senior on 498 has lost an hour debugging because they assumed the robot knew something it didn't. A robot has no common sense — only instructions.
